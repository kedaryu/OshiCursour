# OshiCursour

Windows 10/11 のカーソル一式を、指定間隔でランダムまたは順番に切り替える通知領域常駐アプリです。

## 実装済みの動作

- グループ内からランダム、またはリスト上から順番に切り替え
- 5秒～7日の切り替え間隔（秒・分・時間で入力）
- 自動切り替えの ON / OFF
- 選択中グループのカーソルへ即時切り替え
- 初回起動時に保存したカーソル設定へ復元
- Windows 標準カーソルへ復元
- Windows サインイン時の自動起動 ON / OFF
- 通常はタスクバーへ表示せず、通知領域だけに常駐
- 設定画面を開いている間だけタスクバーへ表示
- × または最小化で設定画面を破棄し、通知領域へ戻る
- 同時に複数起動しない。もう一度起動すると既存アプリの設定画面を開く
- ライトテーマ（初期設定）とダークテーマをヘッダーから切り替え
- 起動時に GitHub Releases を確認し、同意後に更新・再起動
- 更新後の起動に失敗した場合は、更新前の実行ファイルへ自動ロールバック

設定画面は閉じるたびに破棄されます。常駐中に保持するのは通知領域アイコン、設定データ、切り替え用タイマー1個だけです。

初期状態では個人用のカーソルやフォルダーを登録していません。`カーソルグループ` タブからグループとカーソルフォルダーを追加してください。

## 自動検出するファイル名

`.ani` と `.cur` の両方に対応します。ファイル名のキャラクター部分は固定せず、末尾の役割名を判定します。

| Windows の役割 | 主に検出する末尾 |
| --- | --- |
| 通常の選択 | `_通常` |
| ヘルプの選択 | `_ヘルプ` / `_ヘル` |
| バックグラウンドで作業中 | `_バックグラウンド` |
| 待ち状態 | `_待機` |
| 領域選択 | `_領域選択` |
| テキスト選択 | `_テキスト` |
| 手書き | `_手書き` |
| 利用不可 | `_利用不可` |
| 上下に拡大/縮小 | `_上下` |
| 左右に拡大/縮小 | `_左右` |
| 斜めに拡大/縮小 1 | `_斜め1` |
| 斜めに拡大/縮小 2 | `_斜め2` |
| 移動 | `_移動` |
| 代替選択 | `_代替選択` |
| リンクの選択 | `_リンク` |
| 場所の選択 | `_場所` |
| 人の選択 | `_人` / `_人物` |

ファイル名は Unicode 正規化してから比較します。通常カーソルだけは必須です。ほかの役割が不足している場合は、別のカーソルセットが混ざらないよう Windows 標準カーソルで補います。

## 操作

1. `OshiCursour.exe` を起動します。
2. `カーソルグループ` タブでグループとカーソルフォルダーを追加します。
3. `基本設定` でグループ、ランダム/順番、間隔を選びます。
4. `自動切り替え ON` を押します。ON にした直後に1回切り替わり、その後は設定間隔ごとに切り替わります。

自動切り替えを OFF にしただけでは、現在のカーソルを維持します。戻したい場合は次のどちらかを選びます。

- `切り替え開始前に戻す`: OshiCursour を初めて起動した時点の設定へ戻す
- `Windows 標準に戻す`: Windows の標準 Aero カーソルへ戻す

通知領域アイコンの右クリックからも、ON/OFF、グループ変更、即時切り替え、復元、終了ができます。

## ビルド方法

必要環境は Windows 10/11 と .NET 8 SDK、または Visual Studio 2022 の `.NET デスクトップ開発` ワークロードです。

`dotnet.exe` が入っていても、Desktop Runtime だけではビルドできません。コマンドプロンプトで次を実行し、SDKのバージョンが1行以上表示されることを確認してください。

```cmd
dotnet --list-sdks
```

何も表示されない場合は、[.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0) をインストールしてから `build-release.cmd` を実行してください。ダウンロードページでは `SDK 8.0.x` の Windows x64 Installer を選びます。

### 単体で動く EXE

`build-release.cmd` をダブルクリックします。完成品は次に出力されます。

```text
publish\win-x64\OshiCursour.exe
```

GitHub Releases に添付する2ファイルも同時に生成されます。

```text
publish\release-assets\OshiCursour-win-x64.zip
publish\release-assets\checksums.txt
```

.NET ランタイムを含むためファイルサイズは大きくなりますが、別途ランタイムを入れずに起動できます。

## GitHub Releases と自動更新

更新先は公開リポジトリ `kedaryu/OshiCursour` です。アプリは起動時に最新の正式Releaseを1回確認し、アプリより新しい3段階バージョンがある場合だけ更新確認を表示します。確認失敗は起動を妨げず、診断ログだけに記録します。

Releaseには、次の名前のアセットが両方必要です。

- `OshiCursour-win-x64.zip`
- `checksums.txt`

GitHubへソースを反映した後、たとえば `v0.6.1` のタグをpushすると、`.github/workflows/release.yml` がWindows向けEXEをビルドします。ReleaseはDraftとして作成し、必要なアセットを添付してから公開するため、Release immutabilityを有効にしたリポジトリでも利用できます。タグと `CursorCycle.csproj` の `Version` は同じ値にしてください。

```cmd
git tag v0.6.1
git push origin v0.6.1
```

更新処理ではZIPのSHA-256を検証してから、実行中のEXEとは別の一時更新プロセスで本体を置換します。更新後のアプリが正常起動マーカーを作成できなければ、退避した旧EXEへ戻して再起動します。

### ファイルサイズを小さくするビルド

対象 PC に .NET 8 Desktop Runtime が入っている場合は `build-small.cmd` を使えます。出力先は `publish\framework-dependent` です。この形式はフォルダー内のファイル一式を移動してください。

## 保存場所と安全性

アプリ設定と復元用スナップショットは次へ保存します。

```text
%LocalAppData%\OshiCursour\
```

起動しない、または設定画面が表示されない場合の診断ログは次へ保存されます。

```text
%LocalAppData%\OshiCursour\logs\OshiCursour.log
```

`Win + R` を押して次を貼り付けると、ログフォルダーを直接開けます。

```text
%LocalAppData%\OshiCursour\logs
```

カーソル変更と自動起動は、現在の Windows ユーザー用レジストリだけを変更します。管理者権限は要求しません。カーソル適用に失敗した場合は、変更直前の設定へロールバックしてからエラーを通知します。登録したカーソルフォルダーやファイル本体を、アプリから削除する処理はありません。
